JazzBlockKeepr 101

By file location:

A. jazz_elements/jazz_containers.h
B. jazz_elements/jazz_persistence.h
C. jazz_elements/jazz_httpclient.h
D. jazz_elements/jazz_classes.h
E. jazz_functional/jazz_cluster.h
F. jazz_functional/jazz_filesystem.h
G. jazz_functional/jazz_column.h
H. jazz_functional/jazz_dataframe.h
I. jazz_functional/jazz_bebop.h

JazzBlocks can be:

  1. One-shot - Owned by the caller of new_jazz_block()
  2. Volatile - Owned by some JazzBlockKeepr descendant that is not a JazzPersistence (or descendant)
  3. Persisted - Owned by a JazzPersistence (or descendant), typically a JazzSource

Jazz Modules 101

This diagram shows how the different Jazz modules fit. Everything except the dotted nodes is in the repository.

A. Is the folder jazz_elements (all the pieces that work on blocks)
B. Is the folder jazz_functional (all the pieces that support higher level abstractions)
C. Is the folder jazz_main (for the http server only)
F. (not in this repo) Is where front-end developers build, see the REST API reference

You will typically use (5) for extending the back-end in C++. (noted as (x))
You will typically use (4) for connect to other languages, other than R & Python.
You will typically use (3) to run R & Python scripts for both local and remote sources.
You will typically use (2) build docker images containing your applications.

If you use (1), you are the end user, not a developer. ;-)
