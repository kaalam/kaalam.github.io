Detailed Description

The Basic Opcode Pack (BOP)

Snippets: Snippets and code items. (A code item is a minimal snippet.)

Pack: A collection of op-codes.

OpCode: stateless functions and mutators. Category theory provides the necessary composition to manage state without introducing stateless and stateful as separate categories. Only mutators (which introduce addresses) are a separate category.

Formal Fields: The container storing all the fields.

Formal Fields: Organizing op-codes and snippets by Kind.

Cores: The service that owns all the cores.

Core: The execution unit running Snippet objects.

The fundamental collection of op-codes provided in Jazz (any other pack is an uplifted pack).

A core is not a service, it allocates its state in the Volatile container.

This is the one instance of a service containing core to be used in Apis.

Fields are morphisms defined as op-codes and snippets in a unique way for each pair of Kinds.

All fields in a system are managed by this container and searchable by Kind.

These functions are known as op-codes and inherit OpCode. They are implemented in C++ in a pack.

A Pack is a collections of op-codes either the Basic Opcode Pack (BOP) or an uplifted pack.

Functionally, snippets can do the same as op-codes, but they are written in Bebop rather than C++.
