31  * BopCompiler Bebop compiles into bytecode automatically.
32  * A Bebop function is a block of compiled bytecode with its source code stored as an attribute.
33  * Bebop is functional (== functions accept functions as arguments and can return functions), blocks are immutable, reads are safe,
34  writes are idempotent, side effects are not possible.
35  * Bebop source code uses operators (both in the standard core or defined in Bebop) to support “natural expressions”. Operators are
36  only syntactic sugar for functions.
37  * Function overloading is possible with and order and a match for type checking. This provides some metaprogramming, like defining
38  math over many types in one declaration. This also applies to operators (since they are just functions).
39  * Bebop is object oriented. Bebop supports multiple inheritance using combined classes of any number of parents.
40  * Objects can be forked from other objects. A forked object points to its ancestor and is initialized with its state without using
41  extra storing space (until its state changes from that of its ancestor).
42  * A class is internally a keepr that contains variables and functions.
43  * Errors in a wrong mutation can be handled by the ancestor function. Bebop supports alternative functions for managing errors.
44  * Automatic function result caching. Use of "memoize" directive. (Or memoize by default and declare "prevent_result_cache")
45  * Consider 'using ... as ...' the easily create aliases of lvalues and rvalues.
46  * Consider 'matching ... as {... {} ... {} ... {}}' as the case syntax. (Bebop supports match for switching (as in Rust).)
47  * Consider a logic for type assertion (possibly with regex) A bebop function has a low-level way to specify the first, 2nd, ... block
48  passed as arguments. It may also put restriction on types (any numeric sorted, etc.) that throw exceptions when not met. Since
49  refering to the argument as $1, $2, ... is unfriendly, the argument declaration implicitly creates a series of 'using $1 as
50  latitude' statement that allow using the argument with a nice name and an assertion on its type.
51  * Program flow level 1 - 'for each ... {}'
52  * Program flow level 2 - 'if ... {}', 'if not ... {}', 'if ... {} else {}'
53  * Program flow level 3 - 'while ... {}', 'do {} while ...'
54  * Program flow level 4 - 'break', 'fail ...', 'return ...'
