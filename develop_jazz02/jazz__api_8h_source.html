85 /* NOTES from the Jazz draft description including tasks for this
86  --------------------------------------------------------------
87 
88 In Jazz an lvalue is one of:
89 
90 A chain of keeprs abstracted as block ids starting from root ending with the name of a block. They must all exist except, possibly, the last name. If the last name is new, it is created, if it exists, overridden.
91 A data block that will be returned as a Python object, R object or http (GET) resource.
92 In Jazz an rvalue is one of:
93 
94 A block constructor. A constant expression that can be used to build a block from it.
95 Chains of keeprs abstracted as existing blocks starting from root. This includes functions and blocks passed as arguments to functions.
96 A combination of the previous two.
97 A data block that will be passed as a Python object, R object or http (PUT) resource.
98 A delete predicate. This deletes the corresponding lvalue.
99 Since the API has to be REST compatible and is intended for using over a network.
100 
101 All rvalue evaluations are safe. They cannot have side effects. Function calls cannot have side effects.
102 All lvalue assignments are idempotent. Assigning twice has the same effect than assigning once. There is no += operator.
103 
104 GET with a valid rvalue. To read from Jazz.
105 HEAD with a valid rvalue. Internally the same as GET, but returns the header only.
106 PUT with a valid lvalue. To write blocks into a Jazz keepr.
107 DELETE with a valid lvalue. To delete blocks or keeprs (even recursively).
108 OPTIONS with a string. Parses the string and returns the commands that would accept that string as a URL.
109 GET with lvalue=rvalue. Assignment in the server. Similar to “PUT(lvalue, GET(rvalue))” without traffic.
110
